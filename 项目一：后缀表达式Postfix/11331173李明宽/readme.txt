2. 实验心得：
    这次实验中再次深刻学习了Java中的实现中缀表达式转换后缀表达式的过程，并且对编译原理语法分析在实践上有了初步的认识，通过编写错误处理的程序让我认识到了让程序能在各种不同类型的（正确或者各种错误）输入条件下都正常运行不是一件简单的事情，以后在完成项目的过程中也要考虑好对于异常的处理。
    在实现对比尾递归以及循环的性能的时候，除了知道两者效率相当的实验结果，还学会了通过System.setIn以及System.setOut函数重定向输入输出流，把System.out以及System.in重定向到想要的地方。
    第一次相对完整的编写了Javadoc，发现如果能很好的完成Javadoc，能让人更方便理解代码，或者不需要公开源代码就让使用者知道代码怎么使用，以后在完成别的项目的时候也需要注意仔细写Javadoc。


3. 该文件夹的结构：
bin - 文件夹是编译后输出的*.class文件
design -  文件夹里为实验报告，有docx和pdf两个版本
doc - 文件夹里是生成的Javadoc
src - 文件夹里是程序的源代码，其中testStatic文件夹是测试lookahead是否为静态变量的JFrame代码，JUnit文件夹下是JUnit测试代码。
testcases - 文件夹下有13个测试样例，前9个是正确的输入，后4个是错误的输入
build.bat - 编译的脚本
doc.bat - 生成Javadoc的脚本
run.bat - 运行Postfix的脚本
run_JUnit.bat - 运行JUnit的脚本
run_testStatic.bat - 运行测试lookahead是否static的脚本
testcase.bat - 运行测试样例的脚本
